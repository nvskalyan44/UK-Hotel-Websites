A4. Coupon & Promo Codes
Apply discount codes with rich server-side validation
Customers type or tap a promo code in the basket. The code is validated live against the server — any restriction that fails returns a clear, specific error message. Validation happens in the correct priority order so customers always see the most relevant reason for rejection first.
1
Type a code in the promo field and click Apply, or tap a quick-suggestion chip.
2
The server validates: active status → start/expiry dates → eligibility → order type → limits → min order → item/category match.
3
If valid, discount is applied and shown in the total. If not, a specific reason is shown.
4
Tap Remove to clear an applied code at any time.
Validation checkError message shown
Already ordered (first-order-only coupon)"This coupon is for first-time customers only"
Per-customer limit reached"You have already used this coupon the maximum number of times"
Wrong order type (e.g. delivery-only coupon for collection)"This coupon is only valid for delivery/collection orders"
Global use limit reached"This coupon has reached its limit"
Minimum order amount not met"Minimum order of £XX required"
Items don't match category/item restriction"This coupon is not valid for items in your basket"
3-layer validation: Coupons are checked when applied in the basket, again before order placement, and errors surface with the most specific reason first.
3-layer validation First-order-only Category/item restrictions Per-customer limits

A9. Cancel Order NEW
Self-service cancellation within a 15-minute window
Customers can cancel their own order — but only within 15 minutes of placing it and only while the status is still Pending or Confirmed. Once the kitchen begins preparing (status moves to Preparing or beyond), the cancel button disappears.
1
Go to the Account page and expand a recent order.
2
If the order is within 15 minutes and still in Pending or Confirmed status, a red Cancel order button appears.
3
Click the button and confirm the prompt. The order status immediately changes to Cancelled.
4
After 15 minutes, the button disappears. Customers must contact the restaurant directly after this window.
Restriction: Cancellation is server-validated — the API checks both the 15-minute window and the order status, so the rule cannot be bypassed.
15-minute window Pending/Confirmed only Server-validated Confirmation prompt

A20. Delivery Postcode Checker NEW
Real-time postcode validation against admin-configured delivery zones
When a customer enters their delivery postcode at checkout, it is validated against delivery zones configured by the admin. The matching zone's delivery fee is applied automatically. If the postcode is outside all zones, a clear error is shown and the customer cannot proceed.
1
Admin configures zones in Site Settings → Delivery Zones (e.g. prefix "S10" = £2.50, "S11" = free). See B10.
2
Customer enters postcode in the delivery details form. On blur, the postcode prefix (e.g. "S10") is matched against zones.
3
A green check appears for valid postcodes; the delivery fee updates. A red error appears for out-of-zone postcodes — the order button stays disabled.
4
If no zones are configured, all postcodes are accepted (fallback to default delivery fee).

A24. Order Status Email Notifications NEW
Customers receive emails when their order status changes
Whenever an admin changes an order's status (e.g. "Confirmed → Preparing → Out for Delivery"), an HTML email is automatically sent to the customer. Emails include a colour-coded status badge, optional ETA, and a "Track your order" button linking to the live tracking page.
1
Admin updates order status in the Orders panel. A non-blocking email is sent to the customer email address on record.
2
The email shows the new status with a colour-coded badge (green for Delivered, blue for Confirmed, etc.) and the estimated time if set.
3
A "Track your order" button links directly to the live tracking page for that order.
Setup required: Set SMTP_HOST, SMTP_USER, SMTP_PASS, and SMTP_FROM environment variables in the admin site's .env.local to enable email sending. If not configured, emails fall back gracefully to console.log.

B19. Auto-Print Toggle & SMS Notifications NEW
Auto-print new orders and receive SMS/WhatsApp alerts on your phone
Two operational settings have been added to Site Settings: an auto-print toggle (triggers window.print() when a new order arrives) and an SMS alert configuration (sends a text/WhatsApp message to a nominated phone number via Twilio when a new order is placed).
1
Go to Site Settings. Find the Notifications section. Toggle Auto-print new orders on to enable automatic printing whenever a new order is detected.
2
For SMS alerts, toggle SMS/WhatsApp on new order on and enter the restaurant's phone number in E.164 format (e.g. +44 7700 900000).
3
With Twilio configured (TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_FROM), a text message is sent to that number on each new order. Without Twilio, falls back to console.log.
Twilio WhatsApp: Use a WhatsApp-enabled sender number in TWILIO_FROM and format the recipient as whatsapp:+44XXXXXXXXXX to receive alerts on WhatsApp instead of SMS.
